# Cylon Detector field-release workplan

Status: next-stage workplan after the local skunkworks beta. It makes no calendar promises and grants no deployment, vendor, licensing, publication, or research authority.

## Mission

Put the instrument in other people's hands without losing the thing that makes it worth building: every scientific claim stays traceable, every machine output stays provisional, every human call stays visible, and nobody gets handed a fake consciousness score.

The next phase is a release-and-learning problem, not a feature-volume contest. The code already contains the local paper flow, invite boundary, Claim Ledger, portable collaboration, research packaging, and ethics-gated experimental machinery. The work now is to prove that strangers can use those surfaces correctly and that MAC can govern a benchmark without turning disagreement into consensus theatre.

## Operating contract

- Kris owns product decisions and release sequencing.
- MAC review is required only for benchmark versions published under MAC's name.
- No calendar commitments are implied by this plan.
- The generated Vercel project domain is the first hosting target; custom-domain work is deferred.
- Local rehearsal remains public and account-free.
- Hosted analysis remains disabled until its vendor, spending, secret, privacy, and invite gates are separately approved.
- No open-source claim is made until the inherited Atlas licensing boundary is resolved.
- No OSF write, live AI-system intervention, human response collection, or deceptive provenance trial is authorized here.

## Workstream 1: Land a reviewable baseline

**Outcome:** One coherent pull request lands on `master` without losing the preserved alpha or hiding inherited-file changes.

Exit evidence:

- the pull request targets `master`;
- Node 22 CI passes schema, registry, unit, browser, build, dependency, privacy, and secret gates;
- the final diff is reviewed as one product slice;
- the inherited Atlas case-collision cleanup is called out explicitly;
- the branch and `master` are proven synchronized after merge;
- no deployment is inferred from a green merge.

## Workstream 2: Prove the mock field instrument

**Outcome:** A stranger can use a public preview without an account, provider key, invite code, or hosted paper store.

The first preview remains locked to:

```text
MAC_ANALYSIS_MODE=mock
MAC_LIVE_ANALYSIS_ENABLED=false
MAC_INVITE_POLICY=disabled
VITE_USE_ANALYSIS_API=false
```

Field checklist:

1. Load a malformed PDF and see an honest failure.
2. Load a valid paper and understand when it remains local.
3. Run the deterministic gauntlet.
4. Verify a citation against the source page.
5. Revise or reject a machine claim.
6. Save locally, close the tab, reselect the matching PDF, and resume.
7. Export the receipt, Lab Note, Methods report, ledger, stress map, and witness protocols.
8. Import the receipt in a fresh browser and confirm the integrity digest.
9. Delete the local run and all local data.

Exit evidence:

- desktop, mobile, keyboard-only, screen-reader, reduced-motion, and 200 percent zoom passes are recorded against the actual preview commit;
- browser network inspection shows no paper text entering analytics, feedback APIs, or storage services;
- reports remain readable offline and without JavaScript;
- at least one person who did not build the interface completes the path without coaching;
- every confusion becomes a reproducible issue, not an anecdotal redesign request.

Deploying even this mock preview remains an explicit approval gate.

## Workstream 3: Run the first portable MAC rehearsal

**Outcome:** Independent reviewers can disagree through files alone, without accounts or a coordinator database.

The rehearsal uses a permitted or synthetic source and exercises:

- a blind-review packet with honest partial-blinding language;
- locked first calls before provenance reveal;
- a separate provenance envelope;
- second calls and a Provenance Delta;
- at least two independent review contributions;
- a disagreement bundle and three-voice report;
- visible dissent with no averaged score.

Exit evidence:

- packet redaction is manually inspected;
- source and artifact digests bind every exchange to the intended run;
- a coordinator can merge contributions without seeing a hosted reviewer account;
- reviewers can identify where the machine draft, human call, and source evidence diverge;
- the neutral, proponent, and independent voices remain separate in the exported report.

This is a tool rehearsal, not human-subject research. Do not deliberately mislabel provenance or collect study data under this workstream.

## Workstream 4: Establish benchmark governance

**Outcome:** MAC can publish a benchmark version through inspectable Git history rather than chat consensus.

Required decisions:

- named steward for each proposed challenge;
- evidence standard for empirical anchors;
- conflict and recusal expectations;
- minimum review required to move `draft → provisional → published`;
- retirement rules and replacement links;
- correction policy for published versions without mutation;
- changelog and authorship expectations.

Exit evidence:

- one real challenge travels through the GitHub proposal template;
- CI validates its slug, version, five fixed demands, assumptions, confounds, target commitments, anchors, authorship, and change-my-mind observation;
- MAC records the publication decision outside the benchmark bytes;
- the published version is immutable and a proposed change produces a new version.

## Workstream 5: Resolve the release-rights boundary

**Outcome:** Another lab can understand exactly what it may reuse.

Sequence:

1. Approve and send the upstream Atlas license-confirmation request.
2. Record the response and canonical notice.
3. Review the original Cylon Detector path map.
4. Approve Apache-2.0 only for wholly original paths.
5. Preserve inherited notices and mark mixed surfaces honestly.
6. If upstream remains unresolved when a true open-source release matters, extract the bench into a clean repository.

Exit evidence:

- tracked license files and path notices match the documented ownership boundary;
- public copy does not overclaim rights;
- the release can be forked without guessing which license applies to which files.

Sending the request, applying the map, and making an open-source claim remain separate approvals.

## Workstream 6: Prepare a cost-bounded invite pilot

**Outcome:** Hosted analysis can be enabled for a small invited group without creating accounts or retaining scientific content server-side.

Preparation order:

1. Approve the provider, Upstash project, spending ceilings, and human owner.
2. Provision through Vercel Marketplace.
3. Inject secrets through the approved provider and verify only redacted configuration.
4. Create individual HMAC-bound invitations with explicit expiry and quotas.
5. Test invalid, disabled, expired, exhausted, concurrent-use, IP-limit, store-failure, analysis-failure, and global-shutdown paths.
6. Inspect Redis and logs to prove that paper identity, text, digests, findings, citations, and human decisions never entered them.
7. Enable live analysis on a preview before considering production.

Exit evidence:

- local rehearsal still works with Redis and OpenAI unavailable;
- live analysis fails closed whenever quota infrastructure is unhealthy;
- aggregate records contain only the fields in the invite operations runbook;
- each hosted run has exact model and prompt provenance;
- costs cannot exceed the approved pilot boundary without another human decision.

Provisioning, spending changes, secret injection, invitation creation, and live enablement remain explicit gates.

## Workstream 7: Choose the next scientific release deliberately

After the field instrument and governance are proven, choose exactly one lane:

### Evidence release

Demonstrate that the canonical receipt reproduces every report and ledger row exactly.

### Collaborative release

Demonstrate blind/reveal and independent disagreement exchange across separate machines.

### Research release

Run a frozen preregistration and sealed replication, then export an OSF-ready package and RO-Crate without performing an external write.

### Experimental release

Advance an AI indicator protocol or Provenance Flip study only after its named ethics, welfare, consent, deception, debriefing, retention, and publication gates are approved.

Do not run all four lanes in parallel. Pick the one whose evidence would most change the next product decision.

## First decision queue

1. Review and land the current implementation pull request.
2. Decide whether to authorize a mock-only Vercel preview.
3. Nominate one permitted paper or synthetic source for the portable MAC rehearsal.
4. Decide who can approve a MAC benchmark publication.
5. Decide whether to send the upstream license-confirmation request.
6. Leave live analysis and human-subject work off until the field evidence justifies their cost and risk.

## Definition of done

The field-release phase is complete when an uncoached stranger can pressure-test a paper, trace every conclusion, make and revise the human calls, exchange a blind review through files, export a reproducible record, and explain the privacy boundary back to us—and when the project can prove all of that without a score, an account, a hosted paper database, or an implied scientific verdict.
